# Investigation Notes

## Alert Review Process

1. Reviewed the SIEM dashboard.
2. Identified alert severity.
3. Opened the alert details.
4. Reviewed host information.
5. Checked the process name and file path.
6. Reviewed the alert verdict.
7. Documented whether the alert appeared to be a true positive or false positive.

## Important Fields Reviewed

- Time
- Alert name
- Severity
- Status
- Verdict
- Assignee
- Host
- Process name
- Target file
- File hash
- Analyst comments

## Analyst Mindset

When investigating SIEM alerts, it is important to avoid making decisions based only on severity. A critical alert may still need validation, while a lower-severity alert may become important when connected to other suspicious activity.

## Lessons Learned

This project helped me understand the importance of documentation, careful review, and structured investigation when working with security alerts.
